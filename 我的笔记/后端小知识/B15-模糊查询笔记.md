# 模糊查询笔记

* mybatis-plus的分页插件可以放在自己定义的mapper参数中，只能放第一个参数。放完以后默认使用mp自带的分页
* 可以使用post请求接收查询操作，这样可以应付查询参数过多的情况，同时，mapper.xml文件也可以解析json的数据
* `<bind name="patternGroupNm" value="'%'+info.groupNm+'%'"/>`，可以对值进行定义。

### xml文件

```xml
<!--在添加时，根据当前登录信息，分页模糊查询,用户编号位于:AGI.AUR.USR_NO-->
<select id="queryAGWhenAdd" resultMap="BaseResultMap">
    SELECT * FROM (
    SELECT
    *
    FROM
    AUTH_GROUP_INFO AGI
    WHERE AGI.GROUP_ID IN
    (SELECT DISTINCT AUR.GROUP_ID
    FROM AUTH_UGROUP_REL AUR
    WHERE AUR.USR_NO = #{usrNo})
    ) ANS
    <where>
        <if test="info.groupNm != null and info.groupNm != '' ">
            <bind name="patternGroupNm" value="'%'+info.groupNm+'%'"/>
            and ANS.GROUP_NM like #{patternGroupNm}
        </if>
        <if test="info.groupId != null and info.groupId != '' ">
            <bind name="patternGroupId" value="'%'+info.groupId+'%'"/>
            and ANS.GROUP_ID like #{patternGroupId}
        </if>
        <if test="info.groupTp != null and info.groupTp != '' ">
            <bind name="patternGroupTp" value="'%'+info.groupTp+'%'"/>
            and ANS.GROUP_TP like #{patternGroupTp}
        </if>
        <if test="info.dataClass != null and info.dataClass != '' ">
            and ANS.DATA_CLASS = #{info.dataClass}
        </if>
    </where>
    ORDER BY ANS.CREATE_TIME desc
</select>
```

### mapper接口

### 控制层接口

```java
@Operation(summary = "在添加时，根据当前登录信息，分页模糊查询")
@Parameters({
        @Parameter(name = "pageNum", description = "页码", required = true, in = ParameterIn.QUERY),
        @Parameter(name = "pageSize", description = "每页显示数量", required = true, in = ParameterIn.QUERY)
})
@RequestMapping(value = "/query-auth-groups-when-add", method = RequestMethod.POST)
public RestfulResponse<Page<info>, GeneralMeta> queryAGWhenAdd(@RequestParam @Min(1) Integer pageNum, @RequestParam @Min(1) Integer pageSize,@RequestBody info info) {
    return infoService.queryAGWhenAdd(pageNum,pageSize, info);
}
```

# 需求分析

通过走访观察现有的宠物医院，以及从自身作为使用者的角度出发，大致得出本系统的以下几个需求 ：

1. 本系统使用者应该分为三种角色，分别对应用户、医生、管理员，这三种角色应该在系统中行使不同的功能，以至于他们应该具有不同的需求。
2. 用户：用户的需求主要是查看医院和自己的相关的信息，以及向医生发起咨询或求助。同时，考虑到用户要能够更加方便快捷的访问医院的网站，以及能够及时的向医生发起求助。应该给用户在开发一个网页端界面的同时，还应该开发一个微信小程序的界面，以便用户能够随时使用本系统的用户端。
3. 医生：这一角色主要有三个需求，分别是处理挂号单、回答由用户所发起的咨询或求助、以及查看和修改医生的个人信息。需求较为简单，因为这里的“医生”指的只是坐在医务室处理宠物病情的人员，其它的一些事务，比如挂号、录入宠物信息等，由管理员这一角色负责。
4. 管理员：这一角色分为三种身份，分别为**超级管理员**、**普通管理员**、**普通员工**。其中**超级管理员**对应现实医院里的**院长**，应该具有后台管理系统的全部权限。而**普通管理员**则对应现实医院中的**主任**，应具有除了人员任命、科室变动以外的全部权限。最后是**普通员工**，对应现实里的**护士**。只具有部分表的新增权限（诸如挂号、宠物信息录入），而无修改与删除权限。
